# Qleer assignment starter

This repository is a minimal Nuxt starter for hiring assignment.

Qleer builds product compliance workflows for teams that need clear evidence, traceability, and practical risk summaries. This exercise mirrors a small slice of that work: turning structured evaluation results into a concise compliance gap summary.

## Assignment goal

Build a small feature that generates a **Compliance gap summary** from evaluation data.

## What is already provided

- a minimal Nuxt app
- a static evaluation dataset
- shared TypeScript types
- a basic home page showing raw evaluation results
- a stub endpoint at `GET /api/evaluation` that simulates fetching evaluation data

## Candidate task

Improve the summary generation flow end-to-end. You can update UI, server logic, and output structure as needed, while keeping scope pragmatic and maintainable.

## Timebox

Plan for **2 hours**. Prioritize clear decisions over completeness.

## Submission

1. Click **Use this template** to create a new repository from this starter.
2. Do the assignment work in your new repository.
3. Add a `DECISIONS.md` file based on `DECISIONS.template.md`.
4. Share the repository to my github username `@vachmara` or by [email](mailto:hiring@qleer.ai).

## Notes

We'll pay attention to:

- clarity and usefulness of the solution
- quality of reasoning and assumptions
- ability to handle ambiguity
- code readability and structure
- product judgment and tradeoffs
